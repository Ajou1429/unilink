# UniLink 개발 로드맵 v2 — 모바일(필기) × 웹 × 드라이브

> **설계 전환**: 단순 웹 커뮤니티 → **iOS 필기 앱 + 웹 + Google Drive**가 연결되는 AI 학습 파이프라인
>
> **한 줄 컨셉**: 수업 중 굿노트처럼 필기 → 수업 종료 시 AI가 진도·이해도·교수 강조점을 자동 정리 → Google Drive에 저장·구조화 → 시험기간에 그 내용으로 자동 학습계획 수립·코칭

작성일: 2026-06-24 · 대상: 캡스톤 개발 계획서 (v1 로드맵을 대체)

---

## 0. 확정 설계 결정

| 항목 | 결정 | 근거 |
|---|---|---|
| 모바일 | **네이티브 iOS (Swift/SwiftUI + PencilKit)** | Apple Pencil 기반 굿노트급 필기 경험이 핵심 가치 |
| 저장·동기화 | **Google Drive (사용자 본인 계정)** | 대학생 사용률 높고 API 성숙, "각자 드라이브" 구조에 부합 |
| 웹 | **기존 Next.js + Supabase 재사용·확장** | 정리본 열람·학습계획 관리 대시보드로 활용 |
| 백엔드/AI | **Supabase(Auth·DB·메타데이터) + 서버리스 함수 + LLM API** | AI 파이프라인 오케스트레이션 |

## 1. 전체 아키텍처

```
┌─────────────────────────┐        ┌─────────────────────────┐
│   iOS 앱 (SwiftUI)       │        │   웹 (Next.js / Vercel)  │
│  · PencilKit 필기        │        │  · 정리본 열람           │
│  · PDFKit 슬라이드 위 필기│        │  · 학습계획 관리 대시보드 │
│  · 수업 세션 캡처        │        │  · 진도 시각화           │
└───────────┬─────────────┘        └───────────┬─────────────┘
            │  (각자 본인 Drive에 연결)          │
            ▼                                   ▼
        ┌───────────────────────────────────────────┐
        │           Google Drive (사용자 계정)        │
        │   /UniLink/{학기}/{강의}/                   │
        │     ├─ raw/      원본 슬라이드+필기          │
        │     ├─ summary/  진도 정리본                 │
        │     └─ plan/     학습·시험 계획             │
        └───────────────────┬───────────────────────┘
                            │  (메타데이터·트리거)
                            ▼
        ┌───────────────────────────────────────────┐
        │   Supabase  (Auth · Postgres · Storage)    │
        │   + 서버리스 함수 (AI 파이프라인 오케스트레이션)│
        └───────────────────┬───────────────────────┘
                            ▼
        ┌───────────────────────────────────────────┐
        │   AI 계층                                  │
        │   · 손글씨 인식 (PencilKit/Vision/OCR)      │
        │   · LLM (요약·개념·이해도·강조점·계획 생성)  │
        └───────────────────────────────────────────┘
```

**데이터 흐름 (핵심 시나리오)**
1. 수업 전: 강의노트(PDF/이미지) 업로드 → 노트 문서 생성
2. 수업 중: PencilKit으로 슬라이드 위에 필기, 형광펜으로 강조 표시
3. 수업 종료: 세션 마감 → 필기+슬라이드 패키징 → 백엔드로 전송
4. AI 처리: 손글씨 인식 → LLM 분석(5종) → 구조화된 정리본 생성
5. Drive 저장: 정리본·학습계획을 사용자 Drive에 자동 폴더링
6. 시험기간: Drive 누적 내용 기반 자동 학습계획 + 학습 도우미

## 2. 기술 스택

| 계층 | 기술 |
|---|---|
| iOS | Swift, SwiftUI, **PencilKit**(필기), **PDFKit**(슬라이드), Vision(인식), Core Data/Files(로컬), GoogleSignIn + Drive REST API |
| 웹 | Next.js 16, React 19, TypeScript, Tailwind v4, shadcn/ui (기존 자산) |
| 인증 | Supabase Auth + **Google OAuth(Drive scope 포함)** |
| 백엔드 | Supabase(Postgres·Storage·RLS), Edge Function 또는 Next Route Handler |
| 저장 | **Google Drive API**(원본·정리본), Supabase(메타데이터·인덱스) |
| AI | 손글씨/이미지 OCR + LLM API(요약·분석·계획 생성) |
| 배포 | iOS: TestFlight → App Store / 웹: Vercel / DB: Supabase Cloud |

## 3. 단계별 로드맵 (Phase 0 → 8)

전체 기간 예시: **약 12~16주**. 필기 앱·AI 파이프라인이 핵심 난이도. 주차는 권장치.

### Phase 0 — 기획 · 환경 세팅 (1주차)

- [ ] 아키텍처·데이터 흐름·Drive 폴더 규약 확정
- [ ] 계정·키 발급
  - Apple Developer Program (TestFlight 배포용)
  - Google Cloud 프로젝트 + OAuth 동의화면 + **Drive API 활성화**(scope: `drive.file`)
  - Supabase 프로젝트 (region: Seoul)
  - LLM API 키
- [ ] 레포 구조 결정: `unilink-ios`(신규) + 기존 `unilink`(웹) — 멀티레포
- [ ] iOS 프로젝트 생성(SwiftUI) + 웹 환경변수 정리
- [ ] 핵심 가설 1줄 시나리오로 정의 (시연 스크립트 초안)

**산출물**: iOS 빈 앱 빌드 + 웹 구동 + 모든 외부 서비스 연결 확인

---

### Phase 1 — 통합 인증 & Drive 연결 (2주차)

**목표**: iOS·웹이 동일 Google 계정으로 로그인하고 각자 본인 Drive에 연결

- [ ] **Identity 전략**: Supabase Auth를 단일 사용자 신원으로, 로그인은 **Google OAuth**로 통일 → 동시에 Drive 권한 획득
- [ ] iOS: GoogleSignIn → 액세스/리프레시 토큰 확보 → Supabase 세션과 연결
- [ ] 웹: 동일 Google 계정 로그인 → Drive 연결
- [ ] **Drive 초기화**: 최초 로그인 시 `/UniLink/{학기}/{강의}/{raw,summary,plan}` 폴더 자동 생성
- [ ] 토큰 보안: 리프레시 토큰은 서버(Supabase)에 암호화 보관, 클라이언트 최소 노출
- [ ] `drive.file` 스코프 사용으로 앱이 만든 파일만 접근(최소 권한)

**산출물**: 두 클라이언트에서 로그인 → 본인 Drive에 UniLink 폴더 생성 확인

---

### Phase 2 — iOS 필기 앱 코어 (굿노트식) (3~5주차) ⭐핵심

**목표**: 강의노트 위에 Apple Pencil로 필기하는 경험 완성

- [ ] **강의노트 업로드**: PDF/이미지/카메라 → 노트 문서로 가져오기
- [ ] **필기 캔버스**: `PKCanvasView`(PencilKit) + `PDFKit` 합성 — 슬라이드 위 레이어 필기
- [ ] **도구**: 펜·형광펜(강조용)·지우개·올가미·색상/굵기
- [ ] **페이지 모델**: 슬라이드 페이지별 필기 레이어 관리, 페이지 이동/추가
- [ ] **저장 구조**: 로컬(Files/Core Data) + Drive 동기화 (PKDrawing 데이터 + 슬라이드)
- [ ] **제스처·성능**: Apple Pencil 저지연, 손바닥 거치(palm rejection)
- [ ] (선택) 텍스트 박스·이미지 삽입

**산출물**: 슬라이드를 열어 매끄럽게 필기하고 저장·재열람 가능

---

### Phase 3 — 수업 세션 & 데이터 캡처 (5~6주차)

**목표**: "한 번의 수업"을 진도·필기와 묶어 분석 단위로 패키징

- [ ] **세션 개념**: 강의 + 주차(진도) + 날짜를 노트에 연결
- [ ] **수업 시작/종료** UI: 종료 시 "이번 수업 정리하기" 트리거
- [ ] **캡처 항목 구분**
  - 원본 슬라이드(진도 범위)
  - 형광펜 강조 = "교수가 강조한 내용" 후보
  - 추가 필기(여백 메모) = "추가로 필기한 내용"
- [ ] **패키징**: 슬라이드 텍스트 + 필기 레이어 + 메타데이터(강의·주차) → 업로드 페이로드 구성
- [ ] 오프라인 대비: 종료 시 업로드 큐에 적재, 네트워크 복구 시 전송

**산출물**: 수업 종료 한 번으로 분석에 필요한 데이터가 백엔드로 전송됨

---

### Phase 4 — AI 분석 파이프라인 (6~9주차) ⭐핵심

**목표**: 사용자 요구 4단계 중 2단계(자동 정리·분석)를 구현

#### 4-1. 텍스트화 (입력 정규화)
- [ ] 슬라이드 텍스트 추출(PDF 텍스트/이미지 OCR)
- [ ] 필기 손글씨 인식: PencilKit 인식 또는 Vision/외부 OCR (한국어 정확도 검증 필수)
- [ ] 형광펜 영역 좌표 → 해당 슬라이드 텍스트 매핑("강조점" 추출)

#### 4-2. LLM 분석 작업 (5종 산출물)
- [ ] **① 진도 정리**: 해당 주차 강의노트 핵심 요약
- [ ] **② 추가 개념 정리**: 슬라이드에 없는 보충 개념·배경 설명 생성
- [ ] **③ 이해도 분석**: 필기 충실도·누락 구간·오개념 신호 → 이해도 지표/취약 단원 도출
- [ ] **④ 교수 강조 내용**: 형광펜·반복·"시험/중요" 키워드 기반 강조점 정리
- [ ] **⑤ 추가 필기 정리**: 사용자가 여백에 적은 메모를 구조화

#### 4-3. 오케스트레이션
- [ ] 백엔드 함수가 입력 수신 → 작업 분기 → 결과 병합(구조화 JSON)
- [ ] 비용·지연 관리: 작업 묶음 처리, 토큰 한도, 결과 캐싱
- [ ] 처리 상태(대기/진행/완료)를 앱·웹에 표시

**산출물**: 수업 1회 데이터 → 5종 정리본(JSON+가독형) 자동 생성

---

### Phase 5 — Drive 자동 정리 & 학습 계획 (9~11주차)

**목표**: 사용자 요구 3단계(드라이브 자동 작업·계획 정리)를 구현

- [ ] **Drive 자동 저장**: 정리본을 `/summary/{주차}` 에 문서(Docs/PDF/Markdown)로 저장
- [ ] **폴더 자동 구조화**: 강의·주차별 일관된 네이밍·정리
- [ ] **진도 누적 관리**: 주차별 진도·이해도를 Supabase에 인덱싱(빠른 조회용)
- [ ] **학습 계획 자동 갱신**: 새 진도가 쌓일 때마다 "다음에 복습할 것" 계획 업데이트
- [ ] **웹/앱에서 열람**: Drive 정리본 링크 + 진도·이해도 시각화

**산출물**: 수업이 끝날 때마다 Drive에 정리본·계획이 자동으로 쌓이고 갱신됨

---

### Phase 6 — 시험 모드: 자동 계획 + 학습 도우미 (11~13주차) ⭐핵심

**목표**: 사용자 요구 4단계(시험기간 자동 계획·코칭)를 구현

- [ ] **시험 일정 입력/감지**: 과목별 시험일·범위 설정
- [ ] **자동 학습계획 수립**: Drive 누적 정리본 + 이해도 데이터로 일정 생성
  - 이해도 낮은 단원 우선순위
  - 남은 기간에 분산 학습(spaced repetition) 배분
  - 일자별 To-do(복습할 주차·개념)
- [ ] **학습 도우미**
  - 주차별 요약 카드 / 핵심 개념 플래시카드
  - AI 예상문제·셀프 퀴즈로 이해도 점검
  - 진척 체크 → 계획 동적 재조정
- [ ] **웹·모바일 동기화**: 계획·진척 상태 공유

**산출물**: 시험기간에 "오늘 무엇을 공부할지" 자동 제시 + 학습 보조 작동

---

### Phase 7 — 웹 대시보드 연동 (기존 자산 확장) (13~14주차)

- [ ] 기존 Next.js+Supabase에 **Drive 정리본 뷰어** 추가
- [ ] 학습계획·진도·이해도 **대시보드** 구현
- [ ] (기존 시간표/커뮤니티 기능과 연결 — 선택적으로 유지)
- [ ] 모바일에서 만든 데이터를 큰 화면에서 검토·편집

**산출물**: 웹에서 전체 학습 현황을 한눈에 관리

---

### Phase 8 — 통합 · 테스트 · 배포 (15~16주차)

- [ ] **E2E 검증**: 업로드→필기→세션종료→AI정리→Drive저장→시험계획 전 과정
- [ ] **보안**: Drive 토큰 암호화, Supabase RLS, 최소 권한 스코프 점검
- [ ] **비용 관리**: LLM/OCR 호출 한도·캐싱·일일 제한
- [ ] **iOS 배포**: TestFlight 베타 → App Store 심사 대비(권한 사용 설명)
- [ ] **웹 배포**: Vercel + 환경변수 분리
- [ ] **시연 시나리오** 완성 (캡스톤 발표)

**산출물**: 베타 배포 + 발표용 데모

## 4. MVP 경계 (자원 부족 시 우선순위)

| 구분 | 포함 |
|---|---|
| **MVP (반드시)** | 인증·Drive 연결 / iOS 필기(슬라이드 위) / 수업 세션 캡처 / **AI 진도 정리+강조점 추출** / Drive 저장 |
| **차별화 (강력 권장)** | 이해도 분석 / 시험 자동 계획 / 학습 도우미(예상문제) |
| **후순위** | 무한 캔버스·고급 필기 도구 / 실시간 동기화 / 안드로이드 / 커뮤니티 기능 |

> 핵심 임팩트 순서: **필기 → 자동 정리 → 시험 계획**. 이 3개의 수직 흐름 1개를 먼저 end-to-end로 완성한 뒤 품질을 올린다.

## 5. 권장 작업 순서 (의존성)

```
Phase 0 세팅
  └─ Phase 1 인증·Drive
       └─ Phase 2 iOS 필기 ─┐
       └─ Phase 3 세션 캡처 ─┤(필기→캡처 직렬)
            └─ Phase 4 AI 분석
                 └─ Phase 5 Drive 정리·계획
                      └─ Phase 6 시험 모드
                           └─ Phase 7 웹 대시보드
                                └─ Phase 8 통합·배포
```
iOS(2·3)와 백엔드/AI(4) 일부는 더미 데이터로 **병렬 개발** 가능.

| 한국어 손글씨 인식 정확도 | 필기 OCR 오류가 정리 품질 좌우 | 다중 OCR 비교 + 슬라이드 텍스트 우선 활용 + 사용자 교정 UI |

| 토큰·개인정보 보안 | Drive 토큰·학습 데이터 | 서버 암호화 보관, RLS, 최소 권한 |
| App Store 심사 | 권한·데이터 사용 설명 | 카메라/파일/Google 로그인 사용 목적 명시 |
| 오프라인 강의실 | 네트워크 불안정 | 로컬 우선 저장 + 업로드 큐 동기화 |
| 일정 과부하 | 기능 욕심 | MVP 수직 흐름 1개 우선, 후순위 과감히 컷 |

## 7. 즉시 시작 체크리스트 (이번 주)

1. [ ] Google Cloud OAuth + Drive API 활성화, Apple Developer 등록

3. [ ] PDFKit으로 PDF 한 장 위에 필기 합성 PoC
